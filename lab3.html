Lab 3 XML

Task

The task given was to computerize the projects of an organization using XML. I was given required elements like company, projects, clients, team, and more, as well as criteria these elements need like being able to have no or multiple clients for a single project. I started off by analyzing the outlining document before then starting a XML tree diagram.


Diagram

Below (Figure 1) is the tree diagram created from the information provide about what the organization need for this computerization.
Figure 1.


DTD

Once the diagram was created, I proceed with creating a DTD for the XML (Figure 2). A DTD is the layout of rules for an XML object. It is in these files that we can specify what child elements are needed, what content the element has, and what attribute the element needs.
Figure 2.


XML

After creating the diagram and the DTD it was finally time to create the XML file. In it I specified that this was an XML file and was using the DTD I created. After this I created some dummy data to show that the DTD work by creating the element. Below (Figure 3 and Figure 4) you will see the create XML with 2 projects 1 with 2 clients and 2 teams while the other is only a team.
Figure 3.
Figure 4.


Validation

After creating a test XML file, it was time to run it against a validator. I used a validator online called XML Validation. I provided it with my XML file and DTD file and was informed that the XML was valid (Figure 5).
